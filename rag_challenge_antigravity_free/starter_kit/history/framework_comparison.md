# Сравнение фреймворков и бюджетов

## Фреймворки

### 1. LangChain

| ✅ Плюсы | ❌ Минусы |
|----------|-----------|
| Огромная экосистема интеграций | Сложная абстракция, много «магии» |
| Готовые retriever-ы (FAISS, Chroma, BM25) | Частые breaking changes между версиями |
| Удобный chaining и LCEL | Overhead на простых задачах |
| Хорошая поддержка streaming | Debug сложных цепочек — боль |
| Много примеров и документации | Тяжёлые зависимости |

**Лучше для:** быстрого прототипирования, если нужно много готовых интеграций

---

### 2. LlamaIndex

| ✅ Плюсы | ❌ Минусы |
|----------|-----------|
| Заточен на RAG — core use case | Меньше гибкости в кастомных пайплайнах |
| Встроенная обработка метаданных | Менее зрелая экосистема |
| Хороший node/chunk менеджмент | Документация местами неполная |
| Нативная поддержка PDF page numbers | Сложнее отойти от стандартных паттернов |
| Удобный re-ranking pipeline | |

**Лучше для:** RAG-специфичных задач (наш случай), хороший node tracking для grounding

---

### 3. Без фреймворка (прямые вызовы API)

| ✅ Плюсы | ❌ Минусы |
|----------|-----------|
| Полный контроль над каждым шагом | Больше кода для написания |
| Нет лишних зависимостей | Нужно самим реализовать chunking, retrieval |
| Проще debug | Нет готовых интеграций |
| Минимальный overhead | Нет промышленного re-ranking из коробки |
| Легко профилировать TTFT | |

**Лучше для:** максимальной производительности и контроля

---

### 🎯 Рекомендация

**LlamaIndex** — лучший выбор для нашей задачи:
- RAG — его core компетенция
- Встроенный трекинг `page_label` / `file_name` → упрощает grounding
- Готовые re-ranker/hybrid search модули
- Пример из starter_kit уже на LlamaIndex

## Варианты бюджета (на warm-up: 100 вопросов)

### 💰 Экономный (~$0.5–2 за прогон)
- **Embedding**: DeepSeek embeddings (если доступны) или `text-embedding-3-small`
- **LLM**: `deepseek-chat` (текущий, ~$0.14/1M input, $0.28/1M output)
- **Re-ranker**: нет (или BM25 как pre-filter)
- 📊 ~500K input tokens + 20K output = **~$0.08 на LLM за прогон**
- 📊 ~3M embedding tokens = **~$0.06–0.40 на эмбеддинги**

### 💰💰 Средний (~$2–10 за прогон)
- **Embedding**: `text-embedding-3-small` через OpenAI
- **LLM**: `deepseek-chat` + `deepseek-reasoner` для сложных
- **Re-ranker**: LLM-based (дополнительные вызовы DeepSeek)
- Больше top-k, query expansion = больше токенов

### 💰💰💰 Максимальный (~$10–50 за прогон)
- **Embedding**: `text-embedding-3-large`
- **LLM**: GPT-4o / Claude через OpenRouter
- **Re-ranker**: cross-encoder + LLM verification
- Multi-step reasoning, chain-of-thought

> [!TIP]
> С DeepSeek стоимость **очень низкая**. Даже с агрессивным retrieval (top-k=15, query expansion) один прогон warm-up обойдётся в **$0.10–0.50**. Можно экспериментировать свободно.
